lift = normalized hit rate = batch_size * hit_rate

analysis:
   we are using in-batch negatives so there is 1 positive and
   batch_size number of negatives.  With each positive, we have
   batch_size cosine similarity scores for the candidates.
   For randomly trained embeddings each candidate would have an
   equal chance of having the highest similarity score.
   The probability that the positive candidate has the highest
   score is then 1/batch_size.
   So we want out NHR to be better than batch_size*(1/batch_size) = 1

   we would expect:
      healthy learning: 1.9 - 2.9
      strongly optimized: 3.2 - 4.8
      potential data leakage: >= 15

   An NHR > 1 but less than 1.5-ish suggests that the model is
   not quite capturing personalization yet and so more features
   may need to be enabled.
